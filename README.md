# Car Database

A personal encyclopedia of interesting cars. React + Vite, no backend —
**all car data comes from `Car Database.xlsx`** at the repo root, and photos
are plain files in folders. See [VISION.md](VISION.md) for the project vision.

## How updates work

Every push to `main` triggers a GitHub Action that converts the spreadsheet
to JSON, builds the site, and deploys it to GitHub Pages. So:

- **Update car data** → replace `Car Database.xlsx` (locally + push, or via
  GitHub web: open the file → ⋯ → *Upload files* to the repo root with the
  same name). The site rebuilds itself in ~2 minutes.
- **Add photos** → drop image files into `public/images/<brand>/<car-slug>/`
  (see below). No JSON editing needed — the build scans the folders.

## Run it locally

```
npm install
npm run dev        # converts the spreadsheet, then serves on localhost:5173
npm run build      # static site in dist/
```

## The spreadsheet

One row per car. Required columns: `Manufacturer`, `Model`, `Year` — plus the
spec/market/story columns (see existing rows). Rules:

- `Manufacturer` must match a brand name in `public/data/brands.json`
  (currently: Pagani, Ferrari, Koenigsegg, Bugatti, Lamborghini, McLaren).
  Rows for a manufacturer not listed there are skipped with a warning —
  add a brand entry (theme + hero text) to bring a new marque online.
- The car's URL slug is derived from `Model` ("Zonda C12 S" → `zonda-c12-s`).
  Renaming a model changes its slug — rename its image folder to match.
- Spec cells can be numbers or strings ("~2,400 (est.)", "Not published");
  the app displays strings as-is and parses numbers out of them for
  sorting and card stats.

## Adding photos

1. Find the car's slug (it's in the URL: `#/pagani/zonda-f` → `zonda-f`).
2. Put photos in `public/images/<brand>/<slug>/`, numbered `1.jpg`, `2.jpg`…
   — lowest number is the cover/hero. Web resolution (~1600px, quality 85).
3. Add a credit for each photo to `credits.json` in the same folder:
   ```json
   { "1.jpg": "Photo: Name, CC BY-SA 4.0, Wikimedia Commons" }
   ```
4. Push (or upload the files through GitHub web: navigate into
   `public/images/<brand>/`, *Add file → Upload files* — GitHub creates the
   folder if you type `<slug>/1.jpg` as the path).

**Licensing rules** (see VISION.md): publish-safe sources only — Wikimedia
Commons, CC-filtered Flickr/Openverse, own photos, or written permission.
Every photo gets a credit string.

## Generated vs. hand-edited files

| File | Status |
|---|---|
| `Car Database.xlsx` | **source of truth** for car data — edit this |
| `public/data/brands.json` | hand-edited — brand list, themes, hero text |
| `public/data/<brand>.json` | generated by `scripts/build-data.mjs` — never edit |
| `public/images/**` | hand-managed photos + credits |

Favorites are stored in localStorage on each visitor's device.
